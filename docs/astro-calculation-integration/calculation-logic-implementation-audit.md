## Intentionally unavailable modules

The following exact fields are intentionally structured as unavailable and must not be guessed:

- `shadbala.total`
- `kp.significators`
- `varshaphal.varshaLagna`
- `yogini.currentDasha`
- `chara.currentDasha`
- `lalKitab.judgement`
- `sadeSati.dateTable`
- `ashtakavarga.binduMatrix`

## Audit conclusion

The deterministic calculation stack is broadly implemented for the core chart/time/panchanga/varga/dosha/ashtakavarga-total path. The unsupported advanced modules remain intentionally unavailable and are correctly registered as such. The remaining gaps are limited to lower-confidence or partially implemented areas where the evidence corpus does not prove exact parity.
